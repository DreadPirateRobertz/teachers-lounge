TDD Standard & PR Checklist

Melania mandate (cross-rig): tests before implementation. All paths. No happy-path-only PRs. Enforced by Petra at review time.

TDD Checklist — every PR must check all boxes

  • ☐ Tests written alongside or before implementation (not retrofitted)
  • ☐ Happy path covered
  • ☐ At least one error / edge-case test per exported function
  • ☐ Coverage ≥80% on new code (Codecov will report)
  • ☐ No mocking of internal code — only mock at system boundaries (DB, external HTTP, FCM)
  • ☐ Race detector clean (go test -race for Go services)
  • ☐ Lint clean (ruff check / go vet + golangci-lint / eslint)

⚠ PRs with zero new tests for new feature code are rejected without review.

Test Patterns by Layer

Standard patterns for the TL stack. New crew: read this before writing your first test.

Go services (user-service, gaming-service, notification-service)

// Minimal table-driven test pattern
func TestHandlerName(t *testing.T) {
    cases := []struct {
        name    string
        input   RequestType
        want    int // HTTP status
        wantErr bool
    }{
        {"happy path", validInput, http.StatusOK, false},
        {"bad request", invalidInput, http.StatusBadRequest, true},
    }
    for _, tc := range cases {
        t.Run(tc.name, func(t *testing.T) {
            // ... assert
        })
    }
}

DB mocking: use go-sqlmock or in-memory SQLite (modernc.org/sqlite) — never mock production Postgres schema logic away. Redis: use miniredis. HTTP clients: httptest.NewServer.

Cross-rig note (from cfutons): equivalent of cfutons' suppressAuth assertion in TL is verifying the JWT middleware short-circuits with 401 on bad/missing token — assert this explicitly in every handler test that touches an authenticated route.

Python services (tutoring-service, search-service, ingestion-service)

# pytest + httpx TestClient pattern
def test_endpoint_happy_path(client: TestClient):
    resp = client.post("/endpoint", json={"key": "value"})
    assert resp.status_code == 200
    assert resp.json()["field"] == expected

def test_endpoint_validation_error(client: TestClient):
    resp = client.post("/endpoint", json={})  # missing required fields
    assert resp.status_code == 422

DB: use pytest-postgresql or SQLite in-memory. External HTTP: respx for httpx mocks. LLM calls: always mock — never hit real AI gateway in unit tests.

Clock injection (from cfutons): never call datetime.now() directly in business logic. Accept a clock callable parameter so tests can inject a fixed time. This prevents flaky tests on streak/expiry logic.

TypeScript / Next.js (frontend)

// Jest + React Testing Library pattern
describe('ComponentName', () => {
  it('renders correctly on happy path', () => {
    render(<ComponentName prop="value" />);
    expect(screen.getByRole('button', { name: /submit/i })).toBeInTheDocument();
  });

  it('shows error state on failed fetch', async () => {
    server.use(http.get('/api/data', () => HttpResponse.error()));
    render(<ComponentName />);
    expect(await screen.findByText(/error/i)).toBeInTheDocument();
  });
});

MSW: use Mock Service Worker for API mocking — do not mock fetch directly. Playwright E2E (tl-c47): use page.request for auth setup (avoids CSP blocks in dev mode).

Test isolation — module-level state leaks

Pattern to avoid: module-level singletons (caches, counters, timers) that persist across test cases. In Python this means module globals; in Go it means package-level vars; in TS it means module-scope objects not reset in beforeEach.

TL examples at risk: streak counters in gaming-service, funnel state in analytics, session caches in tutoring-service.

Fix: reset or reconstruct stateful objects in each test's setup. For Python: use importlib.reload or extract state into a class. For Go: pass state as a parameter or use t.Cleanup. For TS: call jest.resetModules() or use beforeEach reset.

⚠ Melania flagged this pattern after a funnelTracker regression in cfutons (2026-04-16). Same risk exists in TL streak/spaced-repetition logic.
