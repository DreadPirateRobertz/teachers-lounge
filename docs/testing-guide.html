Smoke Tests

Quick health checks — run after every stack restart.

# All service health probes
curl -s localhost:8000/healthz | jq    # tutoring-service
curl -s localhost:8080/healthz | jq    # user-service
curl -s localhost:8083/healthz | jq    # gaming-service
curl -s localhost:8085/healthz | jq    # analytics-service
curl -s localhost:8001/healthz | jq    # ingestion-service
curl -s localhost:8002/healthz | jq    # search-service
curl -s localhost:9000/healthz | jq    # notification-service
curl -s localhost:3000/api/health      # frontend
Note: All services expose /healthz (liveness) and /readyz (readiness). Liveness checks DB + Redis ping. Readiness additionally checks AI gateway reachable.

TC-01

New session + first message

  1. Register a new student account at localhost:3000/register → redirected to dashboard
  2. Navigate to Chat from the sidebar
  3. Type: "Explain the photoelectric effect" and send → AI tutor responds within ~3s with SSE streaming
  4. Check session created: GET /v1/sessions returns the new session ID
TC-02

Context window + learning style injection

  1. Send 5+ messages in the same session
  2. Use a pronoun reference: "Can you explain that equation again?" → tutor understands the reference (RAG query reformulation active)
  3. Check that the system prompt contains a learning style hint (visible in tutoring-service logs if LOG_LEVEL=debug)
TC-03

Session summary endpoint

  1. After 3+ exchanges, call: GET /v1/sessions/{id}/summary with auth header → returns message_count, avg_response_time_ms, topics list, session_duration_seconds
  2. Verify 403 on another user's session ID
  3. Verify 404 on a nonexistent session ID

TC-10

Mastery gate check

  1. Navigate to Boss Battles from the sidebar
  2. Select a boss chapter → Gate shows mastery percentage across concept paths. Boss is locked if below threshold.
  3. If unlocked, click Enter Battle → Three.js battle scene loads, WebSocket connects (check browser Network tab: WS to localhost:8083/gaming/boss/{id}/stream)
TC-11

Battle flow — answer cycle

  1. Battle starts: boss HP and player HP bars appear → hp event received over WS, both HPs render
  2. A question is displayed. Submit a correct answer → damage event: boss HP drops, combo counter increments
  3. Submit a wrong answer → player HP drops, combo resets
  4. Defeat boss (boss HP reaches 0) → victory screen, XP awarded, loot reveal sequence plays
📸 Screenshot: Loot reveal screen — staggered item reveal animation (LootReveal component), items appearing one by one

TC-12

Boss defeat → XP + notification

  1. After boss defeat, check gaming-service logs for boss_defeats_total Prometheus counter increment
  2. Verify XP awarded: GET /gaming/profile shows updated total_xp
  3. Check notification-service WebSocket push: browser receives boss_unlock event if a new boss was unlocked

Mastery heatmap renders

  2. Verify mastery heatmap loads with concept nodes → colour-coded by mastery level (green high, red low)
  3. Verify "Upcoming Reviews" panel shows spaced-repetition schedule

  2. Return to Adaptive Dashboard → Learning style dial shows Visual/Verbal, Active/Reflective, etc. dimensions
  3. Verify the tutor adapts: send a question in chat → response should match the detected style (e.g. visual learner gets diagram references)

File Ingestion + Search

ingestion-service · search-service · qdrant
TC-40

PDF upload → chunking → search

  1. Upload a PDF via the curriculum upload UI (or directly: POST /ingest/upload with multipart form)
  2. Monitor ingestion-service logs: expect chunk extraction, embedding, Qdrant upsert → Celery task completes, collection updated
  3. Search for content from the PDF: POST /search/query with a relevant query → chunks from the uploaded PDF appear in results
TC-41

Table extraction

  1. Upload a PDF containing a table
  2. Query search-service for table content → chunk with metadata type=table returned, content rendered as markdown table

Unit & Integration Tests

Run all tests locally

Frontend (Next.js / Jest)

cd frontend
npm test -- --watchAll=false    # all Jest tests
npm run lint                     # ESLint
npm run format:check             # Prettier

Python services

cd services/<service-name>
pip install -r requirements.txt
pytest --cov=app --cov-report=term-missing
ruff check .

Go services

cd services/<service-name>
go test ./...
go test ./... -coverprofile=coverage.out && go tool cover -summary=coverage.out
go vet ./...
golangci-lint run
